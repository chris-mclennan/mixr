Or paste the shell installer for your platform:

```sh
# macOS / Linux
curl --proto '=https' --tlsv1.2 -LsSf https://github.com/chris-mclennan/mixr/releases/latest/download/mixr-rs-installer.sh | sh
```

```powershell
# Windows
irm https://github.com/chris-mclennan/mixr/releases/latest/download/mixr-rs-installer.ps1 | iex
```

## Platform details

<CardGrid>
  <Card title="macOS" icon="apple">
    **DMG** — drag `mixr.app` to Applications, then click the dock icon. The launcher opens mixr inside a tmnl tab if tmnl is installed, otherwise Terminal.app.

    Or **`.pkg`** for the CLI install (drops `mixr` on `PATH`). Per-arch — pick Apple Silicon or Intel from the [latest release](https://github.com/chris-mclennan/mixr/releases/latest).

    Requires macOS 10.14 (Mojave) or newer.
  </Card>
  <Card title="Linux" icon="linux">

    ```sh
    curl -LsSf https://github.com/chris-mclennan/mixr/releases/latest/download/mixr-rs-installer.sh | sh
    ```

    For Beatport login you'll need WebKit installed — `apt install libwebkit2gtk-4.1-dev` or the equivalent for your distro.

## Beatport login

On first launch, mixr opens an embedded WebView pointing at `dj.beatport.com`. Sign in there — credentials never touch mixr. The exchanged OAuth token is stored at `~/.mixr/auth.json` and refreshed automatically. Run `mixr --logout` to clear it.

> Audio quality tops out at 256k HLS (AAC). FLAC streams require a higher-tier OAuth scope only available to Beatport's own clients.

## Build from source

```sh
git clone https://github.com/chris-mclennan/mixr
cd mixr
cargo build --release
./target/release/mixr
```

You'll need an audio output stack — CoreAudio on macOS, ALSA / PipeWire on Linux, WASAPI on Windows. All come standard.

### Wrapper script (recommended)

The repo's `run.sh` / `run.ps1` auto-detect librubberband and implement the restart-on-exit-75 loop:

```sh
./run.sh          # macOS / Linux (bash)
./run.ps1         # Windows (PowerShell)
```

## Verify
