## Why a family

These tools were built together, on purpose. They share two things:

**A look.** The same charcoal-bezel, shell-prompt wordmark identity
across both icons — teal `>` for mixr, blue `>` for mnml. Same
monospace headers, same accent-on-prompt theme language in the
running apps.

**An interaction idiom.** Both use the same sectioned settings
overlay (`▸ label: [active] / other  *`), the same keyboard model
(`r` resets a row, `R` resets all, `Enter` saves, `Esc` reverts), and
the same family-bundle launcher chip on macOS.

If you only use one, the family thing is invisible. If you use
both, they cooperate without fighting — same theme file
(`~/.config/mnml/current-theme.toml`), same idiom for settings.

## Install order

Install whichever tool solves the problem you have today. The
other is additive, not a prerequisite.
