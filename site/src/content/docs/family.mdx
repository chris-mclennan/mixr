## Why a family

These tools were built together, on purpose. They share three things:

**A look.** The same charcoal-bezel, shell-prompt wordmark identity
across all three icons — teal `>` for mixr, blue `>` for mnml, orange
`>` for tmnl. Same monospace headers, same accent-on-prompt theme
language in the running apps.

**An interaction idiom.** All three use the same sectioned settings
overlay (`▸ label: [active] / other  *`), the same keyboard model
(`r` resets a row, `R` resets all, `Enter` saves, `Esc` reverts), and
the same family-bundle launcher chip on macOS.

**A way to talk to each other.** `tmnl-protocol` is a tiny
cell-streaming protocol over a Unix socket. tmnl runs as the host;
mnml and mixr can run as native clients (`--blit <socket>`) inside a
tmnl tab. The result: mixr running as a real native tab next to mnml
running as a real native tab next to a plain shell — all in one tmnl
window, all rendered by tmnl's GPU pipeline.

If you only use one tool, the family thing is invisible. If you use
two or three, they cooperate without fighting.

## What works where

- **mixr alone** — runs in any terminal (iTerm2, Alacritty, Ghostty,
  Terminal.app, kitty, WezTerm…). All features work.
- **mnml alone** — same; runs anywhere.
- **mixr inside tmnl** — gets a native tab, with mixr drawing into
  tmnl's GPU surface via `tmnl-protocol`. Full graphics, no nested
  terminal weirdness. Triggered by `mixr.show` from mnml, or
  `:host.launch mixr` from tmnl directly.
- **mnml inside tmnl** — same shape. Native tab, GPU-rendered.
- **mixr panel inside mnml** — the `♪` chip in mnml's statusline opens
  a docked mixr panel inside mnml (file-tree edge across, capped at
  200 cells wide). mnml hosts the panel; mixr runs as the blit
  client. Standalone mnml still works — the panel is just empty until
  you launch it.

## Install order

If you're starting fresh: install whichever tool solves the problem
you have today. The others are additive, not prerequisites.

If you want the full family experience on macOS, install
tmnl first — then mnml and mixr will register themselves as tmnl
launch targets automatically.
